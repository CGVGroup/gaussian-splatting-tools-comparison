This document describes the workflow used for the **LichtFeld Studio** implementation, including:

- application setup (Windows prebuilt)
- dataset loading
- training
- editing and inspection
- export for downstream processing

## 1. Overview

LichtFeld Studio is an all-in-one desktop application for **3D Gaussian Splatting** that integrates training, visualization and editing inside a single GUI environment.

Unlike command-line pipelines, training and rendering are unified: the reconstruction can be inspected and modified interactively during optimization.

The software is a high-performance C++/CUDA implementation designed for real-time reconstruction workflows.

For the experiments a **precompiled Windows binary** was used.

Download from:

After download:

2. Open the `bin` folder
3. Run the executable

No compilation or Python environment is required.

## 4. Input Dataset

LichtFeld Studio requires a **precomputed SfM reconstruction**.

The dataset must already contain camera poses and intrinsics  
(e.g. generated by COLMAP, nerfstudio, Metashape, RealityCapture, OpenSfM).

Preferred format: **COLMAP dataset (undistorted images + point cloud + camera poses)**.

Expected structure:

```text
<location>
|---images
|---sparse
    |---0
        |---cameras.bin
        |---images.bin
        |---points3D.bin
```

The software does not compute Structure-from-Motion internally — it trains Gaussian splats from an existing reconstruction.

## 5. Training

### GUI Training

1. Open LichtFeld Studio
2. Create a new project
3. Import dataset
4. Start training

Training runs interactively inside the application.

During training it is possible to:

- inspect convergence
- navigate the scene in real time
- pause and resume optimization

---

### Command Line Training (Optional)

Basic training:

```bash
LichtFeld-Studio -d data/garden -o output/garden
```

Training with evaluation and rendering:

```bash
LichtFeld-Studio \
    -d data/garden \
    -o output/garden \
    --eval \
    --save-eval-images \
    --render-mode RGB_D \
    -i 30000
```

#### Parameter description

- `-d` → dataset directory
- `-o` → output directory
- `-i` → number of training iterations
- `--eval` → enable evaluation metrics
- `--save-eval-images` → save rendered validation images
- `--render-mode` → rendering mode (RGB, depth, etc.)

---

## 6. Editing and Inspection

LichtFeld Studio provides an integrated scene editing environment.

Available operations:

- select splats (brush / lasso / polygon)
- delete outliers
- crop regions
- transform objects (move, rotate, scale)
- isolate subjects using masks

This allows reconstruction cleaning without external software.

---

## 7. Export

The trained scene can be exported directly from the application.

Output:

```
point_cloud.ply
```

The exported Gaussian splatting file can be used in:

- SuperSplat viewer
- SIBR viewer
- Blender Gaussian splat plugins
- WebGL viewers
